# AI Research Paper Intelligence System

An AI-powered Research Paper Search and Analysis platform that enables users to retrieve relevant research papers using semantic search instead of traditional keyword matching. The application combines modern Natural Language Processing (NLP) techniques with an interactive web interface to simplify the process of exploring research papers.

The system accepts a user query, performs semantic similarity search over precomputed research paper embeddings, retrieves the most relevant papers, and enriches each result with AI-powered analysis including automatic summarization, keyword extraction, named entity recognition, and research paper classification.

---

# Project Overview

Research papers are continuously increasing across every field of Computer Science. Traditional keyword-based search engines often fail to retrieve contextually relevant papers because they depend on exact word matching.

This project solves that limitation by implementing **Semantic Search**, where the meaning of a user's query is compared against the meaning of research paper abstracts using dense vector embeddings generated by Transformer models.

Instead of only returning matching papers, the system also provides:

- AI Generated Summary
- Important Keywords
- Named Entities
- Research Domain Classification
- Similarity Score

making the retrieved papers easier to understand and analyze.

# Key Features

## Semantic Search

Instead of matching keywords, the system retrieves research papers based on semantic similarity.

The user's query and every research paper are converted into dense vector embeddings.

The closest vectors are retrieved using FAISS similarity search.

Advantages

- Better retrieval accuracy
- Context-aware searching
- Faster search using vector indexing
- Handles different wording for the same concept

---

## Automatic Research Paper Summarization

Every retrieved paper contains a lengthy abstract.

Reading multiple abstracts is time-consuming.

The system automatically generates concise summaries using a Transformer model.

This enables users to quickly understand the core contribution of each research paper.

# NLP Pipeline

The core functionality of this project is built around a Natural Language Processing (NLP) pipeline. Instead of relying on traditional keyword matching, the system analyzes the semantic meaning of the user's query and retrieves research papers that are contextually similar.

The complete processing pipeline is shown below.

```
                     User Query
                          │
                          ▼
           Sentence Transformer Embedding
                          │
                          ▼
               FAISS Similarity Search
                          │
                          ▼
            Top-K Relevant Research Papers
                          │
      ┌──────────┬──────────┬─────────────┐
      │          │          │             │
      ▼          ▼          ▼             ▼
 Summarizer  Keywords      NER    Classification
      │          │          │             │
      └──────────┴──────────┴─────────────┘
                          │
                          ▼
                 JSON Response (FastAPI)
                          │
                          ▼
                   React User Interface
```

Every retrieved paper passes through multiple NLP modules before being displayed to the user.

## 1. Sentence Embedding Model

### Model

```
sentence-transformers/all-MiniLM-L6-v2
```

### Library

```
Sentence Transformers
```

### Download Source

https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2

### Approximate Model Size

```
≈ 90 MB
```

### Purpose

This model converts both the user query and every research paper abstract into dense numerical vectors called embeddings.

Unlike traditional keyword search, embeddings capture the semantic meaning of text.

For example,

Query

```
Artificial Intelligence in Healthcare
```

and

Research Paper

```
Deep Learning Techniques for Medical Diagnosis
```

Although the wording is different, both texts have similar meanings.

The embedding model places both vectors close together in the vector space, allowing semantic retrieval.

### Why This Model?

The all-MiniLM-L6-v2 model was selected because

- Lightweight
- Fast inference
- Excellent semantic search performance
- Produces high-quality embeddings
- Widely used in Retrieval-Augmented Generation (RAG) systems

### Role in the Project

```
User Query

↓

Sentence Embedding

↓

FAISS Search
```

## 2. FAISS (Facebook AI Similarity Search)

### Library

```
faiss-cpu
```

### Developed By

Facebook AI Research (FAIR)

### Official Repository

https://github.com/facebookresearch/faiss

### Purpose

FAISS is responsible for performing high-speed similarity search over thousands of research paper embeddings.

Instead of comparing every paper one by one, FAISS indexes the vectors and retrieves the nearest neighbours efficiently.

### Why FAISS?

Without FAISS

```
Query

↓

Compare with every paper

↓

Slow Search
```

With FAISS

```
Query

↓

Indexed Vector Search

↓

Instant Results
```

### Advantages

- Extremely fast similarity search
- Optimized for high-dimensional vectors
- Low memory consumption
- Industry standard vector database library

### Role in the Project

```
Paper Embeddings

↓

FAISS Index

↓

Top Similar Papers
```

```
Hugging Face Transformers
```

### Download Source

https://huggingface.co/facebook/bart-large-cnn

### Approximate Model Size

```
≈ 1.63 GB
```

### Purpose

Research paper abstracts are often lengthy.

Reading multiple abstracts consumes significant time.

The BART Large CNN model automatically generates concise summaries while preserving the meaning of the original abstract.

### Why BART?

BART is one of the most widely used Transformer models for abstractive summarization.

Unlike extractive summarization, which simply copies important sentences, BART generates completely new sentences that capture the important information.

### Advantages

- High-quality summaries
- Preserves semantic meaning
- Fluent natural language generation
- Pretrained on large summarization datasets

### Role in the Project

```
Research Paper Abstract

↓

↓

```
